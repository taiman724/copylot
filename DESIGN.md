# CopyPal — Chrome 拡張設計書

> ページ上のテキスト横にコピーボタンを配置し、転記作業を効率化する汎用 Chrome 拡張

---

## 0. 背景と目的

### 想定する場面
- 1 つのウェブシステムを見ながら、別のウェブシステムにテキストを手動で転記する作業
- 「テキストをなぞって選択 → Ctrl+C → タブ切替 → Ctrl+V」を繰り返す業務
- 1 日数十件レベルで積み重なると目の疲労とミスの温床になる

### このツールがやること
- 表示中のページのテキスト要素の横に **コピーボタン（📋）** を配置
- ボタン1クリックでクリップボードにコピー完了
- 「なぞって選択」が不要になり、作業速度と正確性が向上

### このツールがやらないこと
- 転記先サイトへの自動入力（一切しない）
- ページ内容の外部送信（完全ローカル動作）
- ログイン・認証への介入
- データの保存・蓄積

```
copypal/
├── DESIGN.md                       ← 本書（設計書）
├── ARCHITECTURE.md                 構造・技術・仕組みの解説

├── README.md                       リポジトリ概要
├── extension/
│   ├── manifest.json               権限定義
│   ├── content.js                  ページにボタンを注入する content script
│   ├── content.css                 ホスト span のレイアウト + ハイライトスタイル
│   ├── popup.html / popup.js / popup.css   設定パネル
│   └── icons/icon{16,48,128}.png   拡張アイコン
```

### 構成の特徴
- **content script + popup のみ**で完結。background service worker は **不要**（chrome.storage.onChanged が全 content script に自動配信されるため）
- ネットワーク通信ゼロ。`permissions` にネットワーク系を一切含まない
- 外部ライブラリ不要。素の JavaScript + CSS のみ

## 2. manifest.json（権限設計）

```json
{
  "manifest_version": 3,
  "name": "CopyPal",
  "version": "1.0.0",
  "description": "テキスト横にコピーボタンを配置し、転記作業を効率化",
  "permissions": [
    "activeTab",
    "clipboardWrite",
    "storage"
  ],
  "content_scripts": [
    {
      "matches": ["<all_urls>"],
      "js": ["content.js"],
      "css": ["content.css"],
      "run_at": "document_idle"
    }
  ],
  "action": {
    "default_popup": "popup.html",
    "default_icon": {
      "16": "icons/icon16.png",
      "48": "icons/icon48.png",
      "128": "icons/icon128.png"
    }
  },
  "icons": {
    "16": "icons/icon16.png",
    "48": "icons/icon48.png",
    "128": "icons/icon128.png"
  }
}
```

### 権限の説明（審査向け）
| 権限 | 理由 | リスク |
|---|---|---|
| `activeTab` | 現在のタブのDOMにボタンを注入するため。popup から現タブのホスト名を取得するためにも使用（`tabs` 権限の代わり） | ユーザーがアクティブにしているタブのみ |
| `clipboardWrite` | コピーボタン押下時にテキストをクリップボードに書き込むため | 書き込みのみ、読み取り権限なし |
| `storage` | ユーザー設定（ON/OFF・表示方式・整形 ON/OFF・サイト別無効化リスト）の保存。**業務データは一切保存しない** | ローカル PC 内のみ。同期なし（`storage.local`） |

**不要な権限は一切取らない**: `webRequest`, `tabs`, `host_permissions`, `cookies`, `clipboardRead` 等は使用しない。

> ⚠️ 注意: `<all_urls>` で content script を全ページに注入する設計だが、特定ドメイン限定が求められた場合は `matches` を該当ドメインのみに絞ることも可能。

## 3. コピーボタンの表示方式（3パターン）

### パターン A: 常時表示
- 対象要素の横にコピーボタンを常に表示
- メリット: 迷わない、すぐ押せる
- デメリット: ページがボタンだらけになる可能性

### パターン B: ホバー表示（デフォルト）
- 要素にマウスを乗せたときだけボタンが出現
- メリット: ページの見た目を汚さない
- デメリット: 「ボタンがある」と気づきにくい場合がある

### パターン C: 選択時表示
- テキストをクリック（または軽く選択）するとボタンが出現
- メリット: 意図的な操作でのみ表示される
- デメリット: 1クリック余分に必要

### 実装後の状態
1 つの拡張内に 3 モードすべてを実装し、popup のラジオで切替可能。デフォルトは **B（ホバー）**。`chrome.storage.local` に保存されブラウザ再起動後も保持される。

## 4. content script の設計

### 対象要素の選定

ボタンを付ける対象:
```
td, dd, span, p, li, div, h1〜h6（直接テキストを持つ末端要素）
```

除外条件:
- テキストが空または空白のみ
- テキストが極端に長い（200文字以上 → 段落文章でありコピー対象になりにくい）
- ボタン・リンク・入力要素、`contenteditable` 配下
- 非表示要素（`display: none`, `visibility: hidden`）
- 既にコピーボタンが付いている要素
- **ラベル系タグ**: `<th>`, `<dt>`, `<label>`
- **ラベル系クラス**: `label`, `caption`, `key`, `legend`, `term` のいずれかを持つ要素
- **コロン終端の短いテキスト**（`住所:`, `電話：` など 20 文字以内でコロン終わり）

### ボタンの挙動

```
1. ボタンクリック
2. 対象要素の textContent を取得
3. （オプション）余分な空白を整形
4. navigator.clipboard.writeText() でクリップボードに書き込み
5. ボタンが一瞬 ✅ に変わりフィードバック（0.8秒後に戻る）
6. 画面中央上部にトーストで「コピーしました 〇〇」を 1.7 秒表示
```

### 対象要素のハイライト
ホバー or クリックで対象要素に `data-copypal-active="1"` 属性を付与し、薄い青背景＋枠線で「これがコピーされる」と視認できるようにする。`!important` でページ既存スタイルに勝つ。

### ページへの影響を最小化
- ボタンは **対象要素の子として `appendChild`** で挿入（兄弟挿入だと grid/flex/dl レイアウトを壊すため）
- ボタンと中身のスタイルは Shadow DOM に閉じ込め、ページ CSS と完全隔離
- ボタンの z-index は高く設定するが、モーダル等は邪魔しない
- MutationObserver で動的に追加された要素にもボタンを付与（SPA対応）

### コピー時の整形（オプション）
popup の「余分な空白を整える」が ON のとき、コピー直前に以下を適用:
- 先頭末尾の空白（半角・全角・タブ・改行）を除去
- 内側の改行・タブ・連続する半角空白を **1 個の半角空白** に統一
- **全角空白は保持**（"山田　太郎" の真ん中の全角空白はそのまま）

転記先サイトが「先頭の空白で弾く」「改行混入で入力エラー」のような挙動でも安全に貼り付けられる。

## 5. popup（設定パネル）

実装済みの設定:
- **ON / OFF トグル**: 拡張の有効・無効切り替え
- **このサイトでは無効化**: 現在のサイト（`location.hostname` または `__file__`）でだけ無効にする。`activeTab` 経由で `chrome.tabs.query` し現タブを判定
- **表示方式**: 常時 / ホバー / 選択時 の切り替え
- **コピー時の整形**: 余分な空白を整える ON/OFF（デフォルト ON）

すべての設定は `chrome.storage.local` に保存される。content script は `chrome.storage.onChanged` を監視しており、popup での変更が即座にすべてのタブに反映される。

## 6. 使い方の比較

```
【従来】
元ページ → テキストをなぞって選択 → Ctrl+C → タブ切替 → Ctrl+V → 戻る → 次の項目…

【CopyPal導入後】
元ページ → 項目横の📋ボタンをクリック → タブ切替 → Ctrl+V → 戻る → 次の項目…
```

省略されるのは「なぞって選択」のステップ。これが項目数 × 件数で積み重なると大幅な時間短縮と疲労軽減になる。

---

## 7. 安全性・審査向けの説明ポイント

1. **完全ローカル動作** — 外部通信ゼロ、ネットワーク権限なし
2. **最小権限** — `activeTab` + `clipboardWrite` + `storage` の 3 つのみ
3. **データ非保存** — 取り扱う情報を一切保存・蓄積しない。クリップボードに書き込むだけ。`storage` はユーザー設定（4 個のキー）の保存のみに使う
4. **クリップボード読み取り権限なし** — `clipboardWrite` のみ。書き込み専用で読み取りはできない
5. **ページへの書き込みは最小限** — 対象テキストの末尾にコピーボタン（Shadow DOM ホスト）を挿入するのみ。フォーム入力・送信・削除は一切行わない
6. **ソースコード公開** — 全コードを公開。監査歓迎。外部ライブラリゼロ
7. **Manifest V3** — Google の最新拡張規格に準拠
8. **background service worker なし** — 常駐プロセスを持たない

### iframe 対応について
ページの一部が `<iframe>` 内に表示される構造の場合、現状の `manifest.json` は `all_frames: false` なのでその iframe 内にはボタンが出ない。`true` に切り替えれば対応可能だが、審査時に「全フレームに注入する」と説明が必要になるため、必要が確認されてから変更する。
